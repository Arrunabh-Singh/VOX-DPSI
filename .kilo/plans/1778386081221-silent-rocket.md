# Plan: Add trust proxy setting to server/index.js

## Goal
Add `app.set('trust proxy', 1)` to `server/index.js` after `const app = express()` to properly handle client IP addresses when deployed behind Railway's reverse proxy.

## Change Required
**File:** `server/index.js`

**Current (lines 27-28):**
```js
const app = express()
const PORT = process.env.PORT || 5000
```

**After change:**
```js
const app = express()
app.set('trust proxy', 1)
const PORT = process.env.PORT || 5000
```

## Why This Is Needed
- The application runs on Railway which uses a reverse proxy
- Without `trust proxy`, `req.ip` and `req.protocol` are incorrect (show proxy IP instead of client IP)
- This affects:
  - IP address logging in audit logs (`server/routes/auditLog.js`)
  - Rate limiting accuracy
  - Any middleware that relies on correct client IP (e.g., `req.ip`)

## Implementation Steps
1. Add the single line `app.set('trust proxy', 1)` after line 27 in `server/index.js`
